# Reference Deck Learning

When the user provides internal or prior decks as references, treat them as a system to study, not as a mood board.

## What To Extract

Study the reference decks at three levels:

- deck shell
- page archetypes
- text-image contract

For the deck shell, capture:

- cover style
- section-divider style
- body-page chrome
- page-number or brand-mark behavior
- color restraint and accent rules

For page archetypes, identify recurring layouts such as:

- executive summary / overview
- data or market proof
- comparison matrix
- roadmap or process
- product sample or concept page
- capability / company proof
- closing or decision page

For the text-image contract, ask:

- are images acting as proof, product sample, atmosphere, or decoration
- how close are captions or conclusions placed to the images they explain
- does each page have one dominant visual anchor or many equal-weight objects
- are text blocks stacked, paired, or split across a main panel and a side panel

## What Good Reference Learning Looks Like

The extracted rules should be reusable and behavior-oriented.

Good examples:

- Use one dominant page idea per slide instead of mixing several medium-strength layouts.
- Keep the base palette neutral and let one or two accent colors carry emphasis.
- Separate evidence from explanation by pairing a main visual panel with a tighter insight panel.
- Use timelines, axes, and matrices when the source argument depends on sequence, ranking, or structure.
- Reserve centered hero compositions for product samples or showcase pages, not for every slide.
- Let covers and opening pages breathe through scale and placement, not by forcing the title into an unnaturally narrow column.
- If the deck shell or cover already provides orientation, use the next page to frame stakes or argument shape instead of repeating a full agenda.
- On proof pages, let the main diagram do the proving. Reduce duplicate stat pills, captions, and body copy that merely repeat the same numbers.
- Use whitespace to support contrast and pacing, but pair every large blank field with a strong counterweight so the page still feels held together.

## What To Avoid

- copying brand-specific graphics, logos, or product motifs into unrelated decks
- imitating only the surface colors while missing the underlying page discipline
- applying a product-sample layout to a logic-heavy page
- flattening analytical pages into generic card grids when the references use diagrams or timelines
- stacking bilingual or translated headings into awkward wraps because the original English title was designed too narrowly
- keeping a page only because it exists in the source deck when its job is already handled more elegantly by the cover, shell, or global navigation

## Output

Write `12-reference-study/reference-deck-notes.md` with:

- reference files reviewed
- recurring deck-shell rules
- recurring page archetypes
- text-image rules
- anti-patterns to avoid
- which lessons should transfer to the target deck
